# 截图文案检查示例

本项目不提供 OCR 引擎；但当前 Agent 能可靠读取用户上传截图中的可见文字时，应直接读取并继续检查。本示例也适用于用户已经提供提取文字的情况。

## 上下文

- 产品：航空公司移动端订票页
- 模式：Extract + Review Mode
- 说明：为展示防止过度纠错的规则，本例执行完整审计并列出可保留项

## 批量检查

| 当前文案 | 组件 / 位置 | 问题类型 | 判断 | 推荐文案 | 优先级 | 置信度 |
|---|---|---|---|---|---|---|
| `Login` | 顶部入口 | 需补充上下文 | 如果是动作按钮，使用动词 `Log in`；如果是页面标题、菜单项或名词标签，`Login` 可以保留。 | 按钮：`Log in`；名词标签：`Login` | P2（条件式） | 中 |
| `Depart / Return` | 往返日期选择器 | 轻微润色 | 航旅日期选择器中的常见表达，不应无条件改成 `Departure / Return`。 | `Depart / Return`（保留） | P3 | 高 |
| `Promotion Code` | 表单标签 | 轻微润色 | 当前表达可以理解；消费级移动端可用更短的版本。 | `Promo code` | P3 | 高 |
| `Search` | 航班搜索卡片主按钮 | 轻微润色 | 周围上下文已经明确为航班搜索，当前文案可以保留。独立按钮才更需要说明对象。 | `Search`（保留） | P3 | 高 |
| `Please Select` | 日期字段占位 | 可优化 | 多个字段同时出现时过于宽泛，具体动作更易扫读，但不是高风险问题。 | `Select dates` | P2 | 高 |

## 假设 / 需补充上下文

需要确认 `Login` 的实际组件类型。其余判断基于航空订票页的可见上下文。
