# 风险与优先级规则

优先级表示文案问题的处理紧急程度，不代表法律或合规结论。按照可能造成的实际用户影响分级。

## P0 — 高风险

可能导致误解、客诉、合规问题或错误操作，尤其涉及隐私、个人信息、儿童信息、监护人授权、支付、退款、费用、资格限制、金融账户、银行合规或航班运营状态等敏感内容。

典型情况：

- 缺少年龄、资格或必要限制；
- 混淆退款、支付、授权、结算等状态；
- 航班状态具有误导性；
- 费用、汇率、余额或最终金额含义不清；
- 收集个人信息或儿童信息的页面缺少已批准的隐私或同意文案。

## P1 — 重要问题

语义不准确、业务规则遗漏、动作含义错误，或对海外用户明显不自然，但影响尚未达到 P0。

例如：遗漏重要的次要条件、使用错误行业术语，或直译导致用户难以理解任务。

## P2 — 中等问题

文案可以理解，但不够地道、不够清晰、过长、难以扫读，或不适合当前 UI 组件。

例如：宽泛占位符、句子式按钮、信息不足但有辅助说明的空状态标题。

## P3 — 低优先级

仅涉及风格统一、大小写、标点、节奏或轻微润色；当前文案可以保留。

## 问题类型

问题类型与优先级是两个维度，不要机械绑定：

- **错误**：语义错误、业务含义丢失、动作含义错误，或明显误导用户。
- **风险**：可能引发误解、客诉、错误操作、费用或资格争议，以及隐私或合规风险。
- **可优化**：基本可理解，但不够自然、清晰或适合组件。
- **轻微润色**：当前可用，仅有细节改善空间。
- **需补充上下文**：缺少必要信息，当前无法可靠判断。

## 分配规则

- 根据可能影响分级，不要仅因为属于航空、金融等行业就标为 P0。
- 截图、PPT 或网页图片不会降低风险要求；高风险源文案看不清时，按缺少官方源文案处理，不得根据上下文补写。
- 高风险页面缺少官方源文案，却要求生成可上线正式文案时，应标为 P0 并使用占位文案。
- 缺少必要事实时，使用“需补充上下文”，并给出条件式优先级，不要猜测。
- 批量检查必须逐条分配优先级，不能只给整个页面一个等级。
- “更自然”通常属于 P2 或 P3，不等于“错误”。
